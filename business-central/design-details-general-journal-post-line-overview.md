## <a name="old-architecture"></a><span data-ttu-id="478ae-110">Vanha arkkitehtuuri</span><span class="sxs-lookup"><span data-stu-id="478ae-110">Old Architecture</span></span>  
<span data-ttu-id="478ae-111">Vanhassa arkkitehtuurissa oli seuraavat ominaisuudet:</span><span class="sxs-lookup"><span data-stu-id="478ae-111">The old architecture had the following features:</span></span>  
  
* <span data-ttu-id="478ae-112">Yleisten muuttujien käyttö oli laajaa, mikä lisäsi piilotettujen virheiden mahdollisuutta, jos muuttujia käytettiin väärässä laajuudessa.</span><span class="sxs-lookup"><span data-stu-id="478ae-112">There was extensive use of global variables, which increased the possibility of hidden errors due to use of variables with the wrong scope.</span></span>  
* <span data-ttu-id="478ae-113">Moni proseduuri oli pitkä (yli 100 riviä), mikä lisäsi myös ns. Cyclomatic Complexityn suuruutta (paljon CASE, REPEAT, IF -lauseita) ja teki koodista erittäin vaikean lukea ja ylläpitää.</span><span class="sxs-lookup"><span data-stu-id="478ae-113">There were many long procedures (with more than 100 code lines) that also had high cyclomatic complexity (that is, a lot of CASE, REPEAT, IF nested statements), which made the code very difficult to read and maintain.</span></span>  
* <span data-ttu-id="478ae-114">Useita proseduureja, joita käytettiin vain paikallisesti ja jotka oli tarkoitettu vain paikallisesti käytettäviksi, ei ollut merkitty paikallisiksi.</span><span class="sxs-lookup"><span data-stu-id="478ae-114">Several procedures that were only used locally and were only meant to be used locally were not marked as local.</span></span>  
* <span data-ttu-id="478ae-115">Suurimmalla osalla proseduureista ei ole parametreja eikä käytettyjä yleisiä muuttujia.</span><span class="sxs-lookup"><span data-stu-id="478ae-115">Most procedures had no parameters and used global variables.</span></span> <span data-ttu-id="478ae-116">Osa käytti parametreja ja korvasi yleisiä muuttujia paikallisilla.</span><span class="sxs-lookup"><span data-stu-id="478ae-116">Some used parameters and overrode global variables with locals.</span></span>  
* <span data-ttu-id="478ae-117">Kirjanpitotilien hakujen koodikuvioita ja kirjanpito- ja ALV-tapahtumien luontia ei ole standardoitu ja ne vaihtelevat paikasta toiseen.</span><span class="sxs-lookup"><span data-stu-id="478ae-117">Code patterns for searching the general ledger accounts and creating general ledger and VAT entries was not standardized and varied from place to place.</span></span> <span data-ttu-id="478ae-118">Lisäksi oli paljon päällekkäisiä koodeja sekä asiakas- ja toimittajakoodien välisiä symmetriakatkoksia.</span><span class="sxs-lookup"><span data-stu-id="478ae-118">In addition, there was a lot of code duplication and broken symmetry between customer and vendor code.</span></span>  
* <span data-ttu-id="478ae-119">Suuri osa codeunitin 12 koodista, noin 30 prosenttia, liittyy maksualennus- ja toleranssilaskelmiin, vaikka useissa maissa tai alueilla näitä ominaisuuksia ei tarvita.</span><span class="sxs-lookup"><span data-stu-id="478ae-119">A large part of the code in codeunit 12, approximately 30 percent, related to payment discount and tolerance calculations, although these features are not needed in many countries or regions.</span></span>  
* <span data-ttu-id="478ae-120">Kirjaus, kohdista, peruuta kohdistus, peruuta, maksualennus ja toleranssi sekä vaihtokurssin muutos on liitetty yhteen codeunitissa 12 yleisten muuttujien pitkää listaa käyttäen.</span><span class="sxs-lookup"><span data-stu-id="478ae-120">Posting, Apply, Unapply, Reverse, Payment Discount and Tolerance, and Exchange Rate Adjustment were married together in codeunit 12 using a long list of global variables.</span></span>  
  
### <a name="new-architecture"></a><span data-ttu-id="478ae-121">Uusi arkkitehtuuri</span><span class="sxs-lookup"><span data-stu-id="478ae-121">New Architecture</span></span>  
<span data-ttu-id="478ae-122">[!INCLUDE[prod_short](includes/prod_short.md)] -ohjelmassa codeunit 12 on saanut seuraavia parannuksia:</span><span class="sxs-lookup"><span data-stu-id="478ae-122">In [!INCLUDE[prod_short](includes/prod_short.md)], codeunit 12 has had the following improvements:</span></span>  
  
* <span data-ttu-id="478ae-123">Codeunit 12 on jaettu pienempiin osiin (kaikissa alle 100 koodiriviä).</span><span class="sxs-lookup"><span data-stu-id="478ae-123">Codeunit 12 has been refactored into smaller procedures (all less than 100 code lines).</span></span>  
* <span data-ttu-id="478ae-124">Kirjanpitotilien hakujen vakiomallit on toteutettu käyttämällä kirjausryhmä-taulukoiden apufunktioita.</span><span class="sxs-lookup"><span data-stu-id="478ae-124">Standardized patterns for the search of general ledger accounts have been implemented by using helper functions from Posting Group tables.</span></span>  
* <span data-ttu-id="478ae-125">Kirjausohjelmakehys on toteutettu tapahtumien aloituksen ja lopetuksen hallintaan sekä eristämään luonti kirjanpidosta ja ALV-tapahtumista, ALV-muutosten keräämiseen ja lisävaluutan summien laskemista varten.</span><span class="sxs-lookup"><span data-stu-id="478ae-125">A Posting Engine Framework has been implemented to manage the start and finish of transactions and to isolate the creation to general ledger and VAT entries, the collection of VAT adjustment, and the calculation of additional currency amounts.</span></span>  
* <span data-ttu-id="478ae-126">Koodin kopiointi on poistettu.</span><span class="sxs-lookup"><span data-stu-id="478ae-126">Code duplication has been eliminated.</span></span>  
* <span data-ttu-id="478ae-127">Monet aputoiminnot on siirretty vastaaviin asiakkaan ja toimittajan tapahtumataulukoihin.</span><span class="sxs-lookup"><span data-stu-id="478ae-127">Many helper functions have been transferred to corresponding customer and vendor ledger entry tables.</span></span>  
* <span data-ttu-id="478ae-128">Yleisten muuttujien käyttö on minimoitu, jotta jokainen toimintosarja käyttää parametreja ja kapseloi oman sovelluslogiikkansa.</span><span class="sxs-lookup"><span data-stu-id="478ae-128">The use of global variables has been minimized, so that each procedure uses parameters and encapsulates its own application logic.</span></span>
