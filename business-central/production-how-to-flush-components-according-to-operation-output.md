- <span data-ttu-id="38e60-107">**Varaston arvostus**</span><span class="sxs-lookup"><span data-stu-id="38e60-107">**Inventory Valuation**</span></span>

- <span data-ttu-id="38e60-110">**Varastosaatavuus**</span><span class="sxs-lookup"><span data-stu-id="38e60-110">**Inventory Availability**</span></span>

    <span data-ttu-id="38e60-111">Asteittaista kulutuskirjausta käytettäessä komponenttinimikkeiden saatavuus on paremmin ajan tasalla, mikä on tärkeää kysynnän ja tarjonnan välisen sisäisen tasapainon ylläpitämiseksi.</span><span class="sxs-lookup"><span data-stu-id="38e60-111">With gradual consumption posting, the availability of component items is more up-to-date, which is important to maintain the internal balance between demand and supply.</span></span> <span data-ttu-id="38e60-112">Ilman reitityslinkkien koodeja muut osan vaatimukset voivat olettaa, että se on käytettävissä, niin kauan kuin se odottaa viivästynyttä kulukirjausta.</span><span class="sxs-lookup"><span data-stu-id="38e60-112">Without routing link codes, other demands for the component may believe that it is available as long as it is pending a delayed consumption posting.</span></span>  
- <span data-ttu-id="38e60-113">**Just-in-Time**</span><span class="sxs-lookup"><span data-stu-id="38e60-113">**Just-in-Time**</span></span>

    <span data-ttu-id="38e60-114">Voit mukauttaa tuotteita asiakkaan pyyntöjen mukaan ja siten minimoida jätteet varmistamalla, että työ- ja järjestelmämuutoksia tehdään vain tarvittaessa.</span><span class="sxs-lookup"><span data-stu-id="38e60-114">With the ability to customize products to customer requests, you can minimize waste by making sure that work and system changes only occur when it is necessary.</span></span>  

<span data-ttu-id="38e60-115">Se tehdään yhdistämällä taaksepäin suuntautuvan materiaalinottomenetelmä ja reitityslinkkien koodit siten, että materiaalinottomäärä toimintoa kohden on verrannollinen toiminnon todelliseen tuotokseen.</span><span class="sxs-lookup"><span data-stu-id="38e60-115">You can achieve that by combining backward flushing method and routing link codes so that the quantity that is flushed for each operation is proportional to the actual output of the finished operation.</span></span> <span data-ttu-id="38e60-116">Jos nimikkeiden määrityksessä on käytetty Taaksepäin-materiaalinottotapaa, oletustoiminto laskee ja kirjaa kulutuksen automaattisesti, kun vapautetun tuotantotilauksen tilaksi muutetaan **Valmis**.</span><span class="sxs-lookup"><span data-stu-id="38e60-116">For items that are set up with backward flushing method, the default behavior is to calculate and post component consumption when you change the status of a released production order to **Finished**.</span></span> <span data-ttu-id="38e60-117">Jos määrität myös reitityslinkin koodeja, laskenta ja kirjaus tehdään toiminnon valmistuttua. Toiminnon kuluttama todellinen määrä kirjataan.</span><span class="sxs-lookup"><span data-stu-id="38e60-117">If you also define routing link codes, then calculation and posting occurs when each operation is finished, and the quantity that was actually consumed in the operation is posted.</span></span> <span data-ttu-id="38e60-118">Lisätietoja on kohdassa [Reititysten luominen](production-how-to-create-routings.md).</span><span class="sxs-lookup"><span data-stu-id="38e60-118">For more information, see [Create Routings](production-how-to-create-routings.md).</span></span>
